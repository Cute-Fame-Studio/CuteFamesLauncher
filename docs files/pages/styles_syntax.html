Styles - syntax

In GLC styles file, the information is organized as a JSON data.

Syntax base:

{
...
"target" {
"proriety-1" : "stringValue",
"proriety-2" : 10,
...
"target-child" {
...
}
}
}


Special characters:

In the style process, is important to know the type of element you're targeting. In CSS scripts, the programmer can use keywords like div to referenciate a DIV html tag, a # ti referenciate a element ID or a . to referenciate a class.
This concept also is used by GLC's style language, but with a little modifications:

  • The @ Character have to be used before the identifier to reference a HTML tag.
    Ex.: "@div" will reference all DIV tags in the document.

  • The # Character have to be used before the identifier to reference a ID of a tag.
    Ex.: "#MyUniqueId" will reference the element with id "MyUniqueId".

  • The . Character have to be used before the identifier to reference a CLASS of a tag.
    Ex.: ".SomeClass" will reference all the elements with "SomeClass" as one of it classes.

  • Any parameter witout a identifier character will be headed as an parameter.
    Ex.: "background-color" : "black" will apply the color black to the element that it's encapsulated.
